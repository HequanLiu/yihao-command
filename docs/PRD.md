# 一号指挥台 PRD v4

## 产品定位

**开源 + 本地优先的一人公司管理工具。**

不是另一个Notion/CRM——代码全透明，数据存本地，云同步可选。

---

## 目标用户

- 全栈开发者 / 独立开发者
- SoloPreneur（一人公司）
- 对数据隐私敏感，不信任纯SaaS

---

## 核心功能

| 模块 | 功能 |
|------|------|
| **客户雷达** | 客户信息、跟进记录、往来备注 |
| **日程汇总** | 聚合各平台日历，统一看本周/月 |
| **任务面板** | 看板式任务管理（自己给自己派活） |
| **财务概览** | 收入/支出/利润一张图 |
| **AI助手** | 数据分析、建议生成、自动化流程 |
| **多模态解析** | 统一识别图片/PDF/Excel/Word/语音/文字，自动分类+结构化 |

### 密钥管理

```
用户密钥 = 用户自己生成 + 保存
├── 本地：密钥存在浏览器 IndexedDB（可导出备份）
├── 永不传输到服务器
└── 云端数据：AES-256-GCM 加密后上传，服务器无明文
```

### 数据主权

```
用户数据归属：用户 100% 拥有
├── 本地：SQLite 文件，可直接查看/备份/迁移
├── 导出：JSON / CSV 明文导出，无锁定
└── 删除：卸载 = 删除所有数据，无残留
```

### 技术栈

```
前端：Next.js 14 + Tailwind + shadcn/ui + Tauri v2
本地：SQLite（浏览器端 IndexedDB，桌面端 better-sqlite3）
同步：Cloudflare R2 + S3 SDK（可选付费功能）
加密：Web Crypto API（AES-256-GCM）
桌面：Tauri v2（Rust 后端）
```

## 架构：多模态统一解析层

```
┌─────────────────────────────────────────────────────────┐
│                   MultiModalParser                      │
│                  （通义千问 VL / GPT-4o）                │
└─────────────────────────────────────────────────────────┘
                           │
     ┌──────────┬─────────┼─────────┬──────────┐
     ▼          ▼         ▼         ▼          ▼
  ┌────┐    ┌────┐   ┌─────┐  ┌───────┐  ┌──────┐
  │图片 │    │PDF │   │Word │  │Excel  │  │ 语音 │
  │拍照 │    │扫描│   │合同 │  │预算表 │  │指令  │
  │截图 │    │发票│   │     │  │       │  │      │
  └──┬─┘    └──┬─┘   └──┬──┘  └───┬────┘  └───┬──┘
     │         │         │         │           │
     └─────────┴─────────┴─────────┴───────────┘
                           │
                           ▼
              ┌────────────────────────┐
              │   ParsedDocument       │
              │   （统一结构化输出）    │
              └────────────────────────┘
```

### 支持的输入格式

| 类型 | 来源 | 解析方式 |
|------|------|----------|
| 图片（拍照/截图） | 发票、合同、流水单、发货单 | 通义千问 VL 图片理解 |
| PDF | PO文件、合同、发票 | 提取文字/转图片 |
| Excel | 预算表、排料表 | 读取内容结构化 |
| Word | 合同文本 | 提取文字 |
| 文字 | 自然语言指令 | 直接解析 |
| 语音 | 语音指令 | ASR转文字再解析 |

### 统一输出格式

```typescript
interface ParsedDocument {
  doc_type: 'invoice' | 'contract' | 'budget' | 'po' | 'shipping' | 'bank_statement' | 'unknown'
  confidence: number        // 0-1，识别置信度
  summary: string          // 一句话描述文档内容
  fields: {
    [key: string]: string | number | string[]
  }
  raw_text: string         // 原始文本（留档）
  warnings: string[]       // 可疑字段（需人工确认）
  suggested_action?: 'create_order' | 'match_order' | 'approve' | 'review'
}
```

### 解析流程

```
1. 用户拖入文件 / 上传图片 / 发送语音
2. MultiModalParser 接收
3. 通义千问 VL 识别类型 + 提取内容
4. 返回 ParsedDocument
5. 展示给用户确认 / 修改
6. 确认后进入业务流程（不入库则丢弃）
```

### AI 结果确认机制

```
自动解析 ──▶ 展示确认 ──▶ 用户修改 ──▶ 确认入库
              ▲
              │
         AI 可能有错
         必须人工确认
```
